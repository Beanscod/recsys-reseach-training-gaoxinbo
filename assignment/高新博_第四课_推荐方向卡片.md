## 一、方向选择

### 1. 宽方向

**序列推荐（Sequential Recommendation）**

序列推荐关注用户行为的先后顺序。它不仅考虑“用户过去喜欢过哪些物品”，还考虑“用户按照什么顺序与这些物品发生了交互”，并根据历史交互序列预测用户下一步可能感兴趣的物品。

### 2. 细分方向

**基于 Transformer 的下一物品推荐（Transformer-based Next-item Recommendation）**

目前先重点了解两类经典模型：

1. **SASRec**：使用单向自注意力和因果掩码，根据当前位置之前的行为预测下一物品。
2. **BERT4Rec**：使用双向 Transformer 和掩码物品预测任务，学习用户行为序列的上下文表示。

### 3. 选择原因

- 用户兴趣会随着时间发生变化，行为顺序对推荐结果有直接影响。
- Transformer、自注意力、Embedding 等知识与当前学习内容衔接较好。
- SASRec 和 BERT4Rec 都有公开论文、代码和数据集，适合作为入门复现对象。
- 该方向可以先完成经典模型复现，再逐步关注数据稀疏、长短期兴趣和时间信息等问题。

## 二、研究任务与数据形式

### 1. 具体任务

选择的任务为：**下一物品预测（Next-item Prediction）**。

给定用户按时间排列的历史交互序列：

```text
S_u = [i_1, i_2, i_3, ..., i_t]
```

模型需要预测用户下一次最可能交互的物品：

```text
i_(t+1)
```

例如，某位用户依次观看了：

```text
[《钢铁侠》, 《复仇者联盟》, 《美国队长》]
```

模型需要从候选电影中预测该用户下一部最可能观看的电影，并输出一个 Top-K 推荐列表。

### 2. 原始数据格式

原始交互数据可以使用 CSV 或文本格式，每一行表示一次用户与物品之间的交互：

```text
user_id,item_id,rating,timestamp
1,1193,5,978300760
1,661,3,978302109
1,914,3,978301968
```

主要字段含义：

- `user_id`：用户编号
- `item_id`：物品编号
- `rating`：用户评分或反馈
- `timestamp`：交互时间戳

### 3. 预处理后的序列格式

首先按照 `user_id` 对数据分组，然后根据 `timestamp` 从早到晚排序：

```text
用户1：[1193, 914, 661, 3408, 2355]
用户2：[1357, 3068, 1537, 647, 2194]
```

可以构造如下训练样本：

```text
输入序列：[1193, 914, 661, 3408]
预测目标：2355
```

本次计划优先使用 **MovieLens 1M** 数据集。该数据集包含约 100 万条电影评分，适合进行推荐系统入门实验。

|---|---|---|
| 序列推荐 | Sequential Recommendation | 根据按时间排列的用户行为预测未来偏好 |

## 四、准备了解和比较的方法

### 方法一：SASRec

论文名称：**Self-Attentive Sequential Recommendation**

核心思路：

- 将用户交互过的每个物品转换为 Item Embedding。
- 加入 Positional Embedding，表示物品在序列中的先后位置。
- 使用 Self-Attention 计算历史物品对当前预测的重要程度。
- 使用 Causal Mask，确保预测某个位置时不能看到后面的真实物品。
- 根据用户序列表示与候选物品向量的匹配分数，预测下一物品。

SASRec 的特点是训练可以并行进行，同时能够从较长的历史序列中筛选出与当前预测更相关的行为。

### 方法二：BERT4Rec

论文名称：**BERT4Rec: Sequential Recommendation with Bidirectional Encoder Representations from Transformer**

核心思路：

- 随机将用户历史序列中的部分物品替换为 `[MASK]`。
- 使用双向 Self-Attention，同时利用被掩码位置左侧和右侧的上下文。
- 根据上下文预测原来被遮盖的物品。
- 在测试阶段，可以在用户行为序列末尾添加 `[MASK]`，预测下一物品。

BERT4Rec 与 SASRec 的主要区别是：SASRec 使用从左到右的因果注意力，而 BERT4Rec 在训练时使用双向上下文和 Cloze Task。

### 初步对比计划

| 对比内容 | SASRec | BERT4Rec |
|---|---|---|
| 注意力方向 | 单向、从左到右 | 双向 |
| 训练目标 | 下一物品预测 | 掩码物品预测 |
| 是否使用因果掩码 | 是 | 训练时通常不使用因果掩码 |
| 主要优点 | 结构直接、适合下一物品预测 | 能充分利用序列上下文 |
| 可能困难 | 对序列长度和负采样较敏感 | 训练策略和掩码比例会影响结果 |

本卡片包含 **SASRec、BERT4Rec 两类方法**，满足“至少2类方法”的要求。

## 五、数据划分与评价指标

### 1. 数据划分方式

计划使用常见的 **Leave-one-out** 划分方式。对于每个用户按时间排列后的交互序列：

- 最后一个物品作为测试集目标；
- 倒数第二个物品作为验证集目标；
- 其余交互用于训练。

例如：

```text
完整序列：[A, B, C, D, E]
训练序列：[A, B, C]
验证目标：D
测试目标：E
```

这种方法可以模拟“根据用户已有历史行为预测未来行为”的真实场景，并避免把未来数据泄漏到训练过程中。

### 2. 评价指标一：HR@10

**Hit Rate at 10** 判断真实目标物品是否出现在模型给出的前 10 个推荐结果中。

```text
HR@10 = 命中的用户数量 / 参与测试的用户总数
```

例如，真实物品出现在 Top-10 列表中，则该用户的命中值为 1，否则为 0。

### 3. 评价指标二：NDCG@10

**NDCG@10** 不仅判断真实物品是否被推荐，还考虑真实物品所在的位置。

- 真实物品排名越靠前，NDCG 越高；
- 真实物品未进入前 10 名，NDCG 为 0。

在单个正确物品的下一物品预测任务中，可以理解为：

```text
NDCG@10 = 1 / log2(rank + 1)
```

其中 `rank` 表示真实物品在推荐列表中的排名。

### 4. 可补充指标

后续实验还可以记录：

- `Recall@10`
- `MRR@10`
- 训练时间
- 单轮推理时间
- GPU 显存占用

本卡片明确选择 **HR@10 和 NDCG@10** 作为主要评价指标，满足“至少2个评价指标”的要求。

## 八、当前没有理解清楚的问题

### 核心问题

**当用户交互序列较短、数据比较稀疏时，SASRec 和 BERT4Rec 学到的注意力关系是否足够可靠？怎样同时利用用户的长期偏好和近期兴趣？**

### 目前不清楚的具体内容

1. 用户只有少量历史行为时，自注意力机制是否容易过拟合？
2. SASRec 的因果训练目标与 BERT4Rec 的掩码训练目标，哪一种更适合短序列用户？
3. 长历史序列中存在与当前兴趣无关的旧行为时，模型是否会受到噪声干扰？
4. 是否可以通过时间间隔、对比学习或多兴趣表示改善上述问题？
5. 不同论文的负采样方式、数据划分方式和训练轮数不同，结果是否可以直接比较？
