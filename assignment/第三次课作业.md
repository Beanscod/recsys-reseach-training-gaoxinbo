> 实验目标：完成一次可复现、可检查且不包含密码、SSH 私钥、NAS 凭证或智能体 Token 的 SASRec 快速实验记录。

## 1. 使用路线

- 训练路线：通过 SSH 连接实验室 GPU 服务器，在分配的物理 GPU 上使用 `lesson03` Conda 环境运行。
- GPU 选择：运行前使用 `nvidia-smi` 查看状态；本次只向程序暴露物理 GPU 1，即 `CUDA_VISIBLE_DEVICES=01。屏蔽后，程序内部看到的设备编号为 `cuda:0`。
- 观察方式：第一个终端运行 `watch -n 1 nvidia-smi`，第二个终端运行训练；不结束、不暂停其他用户的进程。
- 复核路线：在当前 Windows 工作区检查数据、代码、日志、摘要、模型文件及敏感信息。复核主机不是本次 GPU 训练主机。

本文不记录服务器地址、账号密码、SSH 私钥、NAS 凭证或任何 Token。

## 2. 文件结构与作用

```text
sasrec_smoke_demo/
├── sasrec_demo.py
├── generate_smoke_data.py
├── README.md
├── data/
│   └── smoke_interactions.csv
├── tests/
│   └── test_sasrec_demo.py
└── outputs/
    └── run_20260715_130458/
        ├── config.json
        ├── train.log
        ├── metrics.jsonl
        ├── summary.json
        └── best_model.pt
```

各部分职责如下：

- `generate_smoke_data.py`：按固定规则生成合成交互数据。数据包含 128 个用户、120 个物品，每个用户 24 次交互，共 3072 条交互；加上表头后为 3073 行。
- `data/smoke_interactions.csv`：固定 Smoke 数据，字段为 `user_id,item_id,timestamp`。
- `sasrec_demo.py`：包含 CSV 读取、ID 映射、按用户留一验证/测试切分、逐位置负采样、SASRec 模型、因果掩码、训练、全物品排序评估、Checkpoint、日志和 GPU 显存监控。
- `tests/test_sasrec_demo.py`：共有 5 项测试，覆盖数据确定性、数据切分与负采样、模型输出形状与因果掩码、一次 CPU 参数更新以及 CPU 端到端产物生成。
- `outputs/run_20260715_130458/`：本次 10 轮 GPU 实验的配置、逐轮指标、日志、汇总和最佳模型。

数据切分时，每个用户前 22 条交互用于训练，第 23 条用于验证，第 24 条用于测试。模型使用物品嵌入和位置嵌入，通过带因果掩码的 Transformer 块编码序列；训练采用正、负样本的二元交叉熵损失，评估使用 HR@10 和 NDCG@10。

## 11. 一个问题及排查过程

问题

在使用 Conda 创建新的虚拟环境时，终端提示需要接受 Anaconda 默认软件源的服务条款，导致环境创建命令无法继续执行。

例如执行：

```bash
conda create -n pytorch python=3.11 -y
```

时，提示尚未接受相关频道的 Terms of Service。

排查过程

首先确认 Conda 使用了 Anaconda 官方默认软件源，其中主要包括：

```text
https://repo.anaconda.com/pkgs/main
https://repo.anaconda.com/pkgs/r
```

这两个频道需要分别接受服务条款。

依次执行以下命令：

```bash
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r
```

也可以使用以下命令接受当前配置中所有需要确认的服务条款：

```bash
conda tos accept
```

随后重新执行环境创建命令：

```bash
conda create -n pytorch python=3.11 -y
```

如需查看当前服务条款的接受状态，可以执行：

```bash
conda tos
```

结论

Conda 创建环境失败的原因不是 Python 版本或环境名称有问题，而是尚未接受 Anaconda 默认软件源的服务条款。

分别接受 `pkgs/main` 和 `pkgs/r` 两个频道的服务条款后，即可正常创建 Conda 虚拟环境。
